# Source Reputation Checklist

Use this checklist when reviewing or extending the demo source set.

- Prefer public source locations or clearly marked synthetic local documents.
- Record a stable `source_id`, source name, URL or filename, source type, and intended usage.
- Do not include credentials, private URLs, internal filenames, or customer-specific references.
- Keep source references broad enough for demo use and avoid implying that the synthetic issuers are real entities.
- Confirm every `source_ids` value in generated position data maps to `Source_Links_Raw`.
